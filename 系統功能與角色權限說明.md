# NPUST 諮商中心資訊系統 — 功能與角色權限說明

> 本文件依據 `index.html` 現行實作整理，作為系統功能與角色權限的內部參考。

---

## 一、系統功能總覽

### 個案管理

| 功能 | 說明 |
|---|---|
| **個案列表** | 主頁；顯示可檢視個案（依權限過濾），可搜尋、篩選學期/狀態、匯出 CSV、新增個案。 |
| **危機個案閱讀** | 專任 / 實習生於危機狀況下輸入姓名/學號/案號＋申請目的，即刻取得該案「當日唯讀」權限，隔日失效；申請與閱讀皆記錄於 `case_access_log.json` 供全員監督。 |
| **不開案個案** | 尚未正式開案的個案清單（諮詢、拒案等）。 |
| **空間預約** | 玉山、雪山、中央山脈…等諮商/會議空間的節次預約（11 節）。 |
| **精神科醫師評估** | 精神科醫師相關評估紀錄頁。 |
| **案號查詢與管理** | 依學期產生案號、查詢既有案號。 |
| **統計分析** | 個案/晤談量、族群、來源等統計。 |
| **事件處理記錄** | 校安事件、危機事件處理紀錄。 |
| **心理測驗資料庫** | psychTestDB — 施測資料歸檔。 |
| **轉銜管理** | 上下學制銜接個案清單（需「轉銜窗口」或管理者權限）。 |
| **身心調適假管理** | 從信箱自動擷取 ml 表單、風險判定、通知承辦；受輔生分頁對所有諮商師開放。 |

### 個案內部功能（進入某案）

- 基本資料、多學期詳情、個管員派任、初次晤談者、來源/主題/BSRS
- 晤談紀錄（含問題評估、後續處遇——皆必填）、伴侶諮商共用（coupleId）
- 附件上傳（拖曳/貼上圖片、PDF、Word）
- 結案評估表、封存 / 解封
- 危機唯讀模式（申請閱讀時顯示唯讀 banner，逐區塊 gate）

### 實習生管理 / 差勤

- **差勤打卡**：地理圍欄軟標記（校區內/外，不阻擋）、手動補登
- **差勤申請**：請假／補休／加班登記／手動打卡；起訖時間自動換算時數；附件；即時剩餘額度
- **差勤管理後台**：三分頁（打卡總覽 / 假別與額度 / 審核）；假別可完整編輯（名稱/扣抵/預設額度/期間）；暑休 7/1 自動回滿 40hr；加班補休認證制
- **未申請人員通知**、**行事曆檢視**

### 系統管理

- **使用者管理**：新增/編輯/停用帳號、指派角色、附加身分、allowedCases、批次匯入
- **系所學制對照**、**緊急聯絡人快速選項**
- **資源回收桶**：軟刪除個案還原
- **未歸屬記錄**：晤談紀錄無主檢查

### 設定與工具

- **錯誤回報 / 許願池**：全員可提意見，含燈號通知
- **PDF 轉圖片**
- **稽核日誌**：所有寫入操作永久留存
- **偏好設定**：PIN、side bar 拖曳排序、輔導人員頻率排序
- **版本更新紀錄**：dev / prod 兩個分頁

## 二、角色權限對照

系統角色分兩層 —— **主角色**（role，9 種）＋**附加身分**（extraRole 或 flag，可疊加）。

### 主角色（ROLES）

| 角色 | 可看個案範圍 | 特色權限 |
|---|---|---|
| **系統管理者** | 全部 | 保留角色，通常搭配「管理者」附加身分。 |
| **主任** | 全部 | 內建等同管理者：使用者管理、稽核、全案可見、全站編輯、危機閱讀監督區可見。 |
| **專任社會工作師 / 專任諮商心理師 / 專任臨床心理師** | 主責 + 個管 + 初次晤談者 + 未派案 | 可申請「危機個案閱讀」；可看監督區 feed；可指派其他人為個管員（於自己主責的案）。 |
| **兼任諮商心理師 / 兼任臨床心理師** | 主責 + 個管 + 初次晤談者 | 一般晤談與紀錄；不可申請危機閱讀；不可看監督區。 |
| **實習諮商心理師** | 主責 + 個管 + 初次晤談者 | **獨有**：差勤打卡、差勤申請（請假/補休/加班/手動補登）；可申請危機閱讀，但**看不到監督 feed**（被監督者）。 |
| **義務輔導老師** | 主責 + 個管 + 初次晤談者 | 一般晤談紀錄使用（下拉選單獨立分組）。 |
| **駐校精神科醫師** | 主責 + 個管 + 初次晤談者 | 精神科評估頁主要使用者。 |

> **可見性核心規則**（非管理者）：`_caseVisibleToUser` = 主責 ∨ 個管員（allowedCases）∨ 初次晤談者 ∨ 未派案 ∨ 已封存。

### 附加身分（可疊加於任何主角色）

| 附加身分 | 加獲權限 |
|---|---|
| **管理者**（`isAdmin:true`） | 等同主任：全案可見、使用者管理、資源回收桶、未歸屬記錄、統計全開、危機閱讀監督區。 |
| **個案管理員** | 於 `allowedCases` 指定個案可讀寫，即使不是主責。 |
| **實習生行政督導** | 差勤管理後台審核權、額度設定、看實習生打卡；本身可打卡/申請（驗證用）。 |
| **實習生專業督導** | 同上，走專業線審核。 |
| **轉銜窗口**（`isTransferContact`） | 開啟側邊「轉銜管理」頁。 |
| **身心調適假窗口**（`isMentalLeaveContact`） | 可主動從信箱擷取 ml；管理頁可看主責/全部；受輔生分頁完整檢視。 |

### 功能 × 角色可見矩陣（側邊欄）

| 功能 | 主任/管理者 | 專任 | 兼任 | 實習生 | 義輔/精神科 |
|---|:---:|:---:|:---:|:---:|:---:|
| 個案列表 / 案號 / 統計 / 精神科評估 / 空間預約 / 事件記錄 / 心測資料庫 | ✅ 全案 | ✅ 有權案 | ✅ 有權案 | ✅ 有權案 | ✅ 有權案 |
| 危機個案閱讀 | ✅ 監督 | ✅ 申請＋監督 | ❌ | ✅ 僅申請 | ❌ |
| 轉銜管理 | ✅ | 需窗口 | 需窗口 | 需窗口 | 需窗口 |
| 身心調適假管理 | ✅ | ✅（受輔生分頁） | ✅ | ✅ | ✅ |
| 差勤打卡 / 差勤申請 | ✅（驗證用） | ❌ | ❌ | ✅ | ❌ |
| 差勤管理後台 | ✅ | 需督導身分 | 需督導身分 | ❌ | ❌ |
| 使用者管理 / 回收桶 / 未歸屬記錄 | ✅ | ❌ | ❌ | ❌ | ❌ |
| 錯誤回報 / PDF 工具 / 稽核 / 偏好 / 版本紀錄 | ✅ | ✅ | ✅ | ✅ | ✅ |

### 特殊限制

- **非管理者啟動**只下載自己有權檢視的個案（資料最小化 + 防呆），不會意外覆蓋全體共享檔。
- **危機閱讀**當日到期；閱讀動作 session 節流寫 log；實習生看不到監督 feed，但**每筆閱讀仍被記錄**供主任/專任檢視。
- 主任 / 管理者本身就全案可見，故側邊欄「危機個案閱讀」不顯示。
